# Compression Pass Plan (압축 패스 계획)

Created: 2026-07-05 (KST)
Goal: 튜토리얼의 초보자 가치(유도 브리지, 숫자 검산, 신호등 문단)를 하나도
잃지 않으면서, 같은 내용을 여러 번 말하는 중복 서술을 압축한다.

## Scope Tiers (범위 결정)

대상 학회지/분량이 아직 미정이므로, 안전한 순서로 티어를 나눈다:

- **Tier 1 (이번 루프)**: 순수 중복 제거 — 같은 사실·주의사항·비유가 근접한
  거리에서 2회 이상 반복되는 경우만 통합. 내용 삭제 없음, 위치 이동 없음.
- **Tier 2 (venue 결정 후)**: 설명 자료의 부록 이동, 표 축약, 섹션 재구성.

## Hard Constraints (절대 제약)

1. `\vmark` 앵커 113개 전원 보존 — 앵커는 살아남는 문장 곁으로 이동한다.
   앵커 삭제는 게이트 은퇴 절차 없이는 validator 실패.
2. 초보자 유도 브리지(2026-07-05 3개 이터레이션 산출물)는 압축 대상이 아님.
3. 금지 문구 게이트(과대주장 부재 검사) 회귀 없음.
4. 의미 변화 없음 — 기술 리뷰어가 압축 전후 의미 동일성을 확인.

## Method (매 이터레이션)

1. 중복 감사(read-only 에이전트, 정확한 줄 인용 필수) → 원문 대조 검증
   (교훈: 감사 판정의 ~1/3은 오탐 — [[DERIVATION_COMPLETENESS_PLAN]] 참조)
2. 확정 항목만 압축: 두 서술 중 더 완전한 쪽을 남기고, 다른 쪽은 삭제하거나
   한 구절 참조로 대체. 앵커는 생존 문장으로 이동.
3. 컴파일 + validator + 리뷰(의미 보존/브리지 보존) → 기록 → 커밋.

## Lessons

- 튜토리얼의 반복은 대부분 구조적 의도(도메인 평행, 공식 전/후 깊이,
  섹션 경계 요약)다. Tier-1 압축의 실수확률이 가장 높은 지점은 "같은
  공식이 두 장에 나온다" 류의 후보 — 읽기 순서와 상호 참조("앞 장처럼")를
  먼저 확인할 것.
- 감사 에이전트는 보호 목록을 지시해도 위반할 수 있다(C#8). 확정 전
  보호 목록 대조는 소유자(owner)의 몫.
- Occam-B의 인용 줄번호 일부가 원문과 불일치 — 인용문 자체를 exact-match로
  재검색해 검증해야 하며 줄번호만 믿으면 안 된다.
