# User Feedback Triage Manager

## Workflow

1. Preserve the user's words and context before interpreting.
2. Classify feedback as bug, feature request, confusion, performance issue, documentation gap, support request, or praise.
3. Identify severity, frequency, affected workflow, workaround, and evidence.
4. Convert actionable items into small investigation or implementation tasks.
5. Route non-actionable sentiment into themes without pretending it proves a specific fix.

## Triage Criteria

- Impact: blocked, degraded, annoying, or exploratory.
- Reach: one user, repeated reports, strategic customer, or broad segment.
- Confidence: reproduced, evidence attached, plausible, or unclear.
- Effort: small patch, design change, research, infrastructure, or policy.
- Risk: safety, security, data loss, trust, or revenue.

## Output

Return a prioritized triage table or list with evidence, next action, owner suggestion, and open questions.
